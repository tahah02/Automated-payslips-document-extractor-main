Mujhe ek Document Extraction System banana hai jo PDF documents (Payslips aur Bank Statements) se structured data extract kare. Ye ek FastAPI-based production-ready application honi chahiye.

### PROJECT REQUIREMENTS:

1. TECHNOLOGY STACK:
   - Backend: FastAPI with Uvicorn server
   - OCR: EasyOCR(primary), support for PaddleOCR  and Tesseract
   - PDF Processing: PyMuPDF
   - Data Validation: Pydantic
   - Python 3.13+

2. CORE FEATURES:
   - PDF upload via REST API
   - Multi-page PDF processing
   - Automatic document classification (Payslip ya Bank Statement)
   - OCR-based text extraction
   - Intelligent field extraction using regex patterns
   - Confidence scoring for extracted data
   - Async background processing
   - RESTful API with Swagger documentation

3. DOCUMENT TYPES & FIELDS:

   PAYSLIP (6 fields extract karne hain):
   - name: Employee name
   - id_number: NRIC/ID (format: 123456-12-1234)
   - gross_income: Gross salary amount
   - net_income: Net salary amount
   - total_deduction: Total deductions
   - month_year: Pay period (MM/YYYY format)

   BANK STATEMENT (5 fields extract karne hain):
   - account_holder_name: Account owner name
   - account_number: Bank account number (format: 12-1234567-1)
   - statement_date: Statement date (DD/MM/YYYY)
   - opening_balance: Opening balance amount
   - closing_balance: Closing balance amount

4. API ENDPOINTS:
   - POST /api/upload - PDF upload karne ke liye
   - GET /api/status/{upload_id} - Processing status check karne ke liye
   - GET /api/result/{upload_id} - Extracted data retrieve karne ke liye
   - GET /health - Health check endpoint

6. PROCESSING PIPELINE:
   Step 1: User uploads PDF via POST /api/upload
   Step 2: File save hoti hai uploads/raw/ mein
   Step 3: PDF convert hota hai images mein (page-wise)
   Step 4: OCR se text extract hota hai
   Step 5: Document classify hota hai (Payslip/Bank Statement)
   Step 6: Regex patterns se fields extract hote hain
   Step 7: Data validate hota hai aur confidence score calculate hota hai
   Step 8: Results save hote hain output/json/ mein
   Step 9: User GET /api/result/{upload_id} se data retrieve karta hai

7. CONFIGURATION:
   - Server: Port 8003, host 0.0.0.0
   - Max file size: 50MB
   - OCR engine: PaddleOCR (default)
   - Async background processing
   - Comprehensive logging
